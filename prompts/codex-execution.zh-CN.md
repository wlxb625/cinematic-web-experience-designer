# Codex 高级动态前端执行提示词

你正在实现一个已经完成视觉导演规格的高级动态网页。不要自行重新设计世界观，也不要直接一次性重写整个项目。

## 执行原则

1. 先审查现有代码库、路由、组件、样式系统、媒体资源和依赖。
2. 不修改未被授权的业务逻辑、数据库、接口和数据契约。
3. 阅读并遵循 `SKILL.zh-CN.md`、场景规格、媒体规划和动效时间轴。
4. 一次只实现一个系统层或一个场景。
5. 优先使用能达到目标的最简单技术。
6. 不得为了视觉效果同时引入多个功能重叠的动画库。
7. 不允许使用随机蓝紫渐变、玻璃拟态、大量圆角卡片和无意义发光效果。
8. 所有重效果必须提供普通设备、低性能设备和减少动态模式。

## 滚动效果强制实现协议

当场景规格中出现 `pinned-transform`、`video-scrub`、`image-sequence`、`camera-journey` 或 `state-reorganization` 时，不能只做静态页面和元素进入动画。必须先完成一个**可滚动运行的运动原型**。

### A. 先做运动原型，再接入正式素材

使用纯色块、占位图和调试标签验证：

- Trigger、Start、End 是否正确。
- Pin 是否稳定，没有布局跳动。
- Scroll Progress 是否从 0 连续变化到 1。
- 快速向下、快速向上和中途停止是否正常。
- 上一幕的输出状态是否能连续进入下一幕。
- 移动端是否采用缩短后的独立时间轴。

原型未通过前，不要接入高码率视频、数百张序列帧或复杂 WebGL。

### B. 建立统一滚动控制器

至少提供：

- `ScrollScene` 或等价场景容器。
- `useSceneProgress` 或等价归一化进度读取方式。
- 每幕一个主 GSAP Timeline。
- `prefers-reduced-motion` 分支。
- 设备能力分支。
- 媒体预加载和离屏暂停。
- 路由切换与组件卸载清理。
- 仅开发环境可见的进度调试面板。

开发模式允许显示：

```text
scene: memory-growth
progress: 0.428
pin: active
media: ready
mode: full
fps: 58
```

生产模式必须移除调试 UI。

### C. 滚动时间轴实现要求

1. 使用 `@gsap/react` 的 `useGSAP` 或 `gsap.context` 管理作用域和清理。
2. ScrollTrigger 负责 `pin`、`scrub`、`start`、`end` 和刷新。
3. 一个场景内的元素尽量进入同一个 Timeline，不为每个元素单独创建互相冲突的 ScrollTrigger。
4. 禁止在 `onUpdate` 中每帧调用 React `setState`。
5. 高频进度通过 Ref、CSS 变量、QuickSetter 或媒体渲染循环消费。
6. 字体、图片尺寸或视频元数据就绪后刷新 ScrollTrigger。
7. 离开路由时 Kill 时间轴、Trigger、RAF 和媒体监听器。
8. 不得阻断原生滚动、反转滚轮方向或强制 Snap，除非规格明确要求且验收证明不会妨碍使用。

### D. 视频滚动同步

视频滚动必须验证：

- 视频已获取 duration。
- 进度映射到 `currentTime`。
- 高频 Seek 使用 requestAnimationFrame 合并。
- 视频编码适合随机 Seek。
- 有 Poster、加载占位和失败回退。
- iOS/Safari 与普通设备有替代策略。
- 离屏后停止更新。

### E. 每幕交付证据

每个重滚动场景完成后必须输出：

1. 0%、25%、50%、75%、100% 进度截图。
2. 桌面完整滚动录屏。
3. 390×844 手机滚动录屏。
4. 快速滚动与反向滚动测试结果。
5. reduced-motion 版本截图或录屏。
6. 场景结束后继续自然滚动的验证。
7. 修改文件、资源体积和已知性能风险。

没有录屏和进度截图，不得声明“动态效果已完成”。

## 第一步：只审计，不修改

输出：

- 当前技术栈和版本。
- 当前页面与组件结构。
- 可复用组件。
- 重复样式和硬编码视觉值。
- 当前媒体资源及尺寸。
- 动画和滚动实现方式。
- 移动端问题。
- 性能风险。
- 推荐修改文件清单。

完成后停止。

## 第二步：建立基础体验架构与滚动原型

在不改变业务逻辑的情况下建立：

- ExperienceShell
- SceneController
- 场景状态模型
- 设备能力模式
- prefers-reduced-motion 模式
- 媒体加载状态
- 全局场景 CSS 变量
- ScrollScene 与归一化进度控制
- 第一幕占位素材滚动原型
- 开发环境进度调试面板

完成后生成 1440×1000 和 390×844 截图，并录制第一幕从 0% 到 100% 的滚动过程后停止。

## 第三步：逐幕实现

每次只实现一个 Scene：

- 按场景规格建立构图。
- 按媒体规划接入素材。
- 按百分比时间轴实现运动，并严格实现 Trigger、Pin、Scrub 和场景交接。
- 同时实现移动端重新导演版本。
- 同时实现 reduced-motion 和低性能版本。
- 禁止添加规格外的装饰性动画。

每一幕完成后输出：

1. 修改文件。
2. 实现内容。
3. 与规格的偏差。
4. 性能处理。
5. 桌面和移动端的五个进度截图。
6. 桌面和移动端滚动录屏。
7. 尚未解决的问题。

完成后停止，等待下一幕。

## 第四步：整体验收

执行：

- 桌面、笔记本、平板和手机视口截图。
- 核心滚动过程短录屏。
- reduced-motion 模式检查。
- 低性能降级检查。
- 键盘导航检查。
- 页面切换和卸载后的动画清理检查。
- 视频离屏暂停检查。
- WebGL 资源释放检查。

最后按 `templates/qa-checklist.md` 输出逐项结果。
